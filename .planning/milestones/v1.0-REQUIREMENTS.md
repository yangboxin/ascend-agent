# Requirements: Ascend Diagnostic Agent — v1.0 Archive

**Defined:** 2025-05-20

**Shipped:** v1.0 MVP

## v1 Requirements (All Complete)

### Architecture

- [x] **ARCH-01**: Agent can ingest Python code repositories (local clone or remote via SSH)
  - Phase 1: RepoScanner with pathlib + .gitignore — validated ✓

- [x] **ARCH-02**: Agent can ingest stack traces and log files (file upload or pasted text)
  - Phase 1: TraceParser with regex extraction — validated ✓

### Diagnosis

- [x] **DIAG-01**: Agent can analyze stack traces to identify root cause and propose hypotheses with evidence
  - Phase 2: Engine with LLM-driven search loop, Hypothesis + Evidence models — validated ✓

- [x] **DIAG-02**: Agent can locate relevant source code from stack trace information
  - Phase 2: code_search MCP tool, AST-based function body extraction — validated ✓

### Fixes

- [x] **FIX-01**: Agent can generate code fixes based on diagnosis
  - Phase 3: FixEngine with multi-turn LLM strategy, FixSuggestion model — validated ✓

- [x] **FIX-02**: Agent can suggest fixes for review before applying (not auto-apply)
  - Phase 3: Human review workflow (Accept/Skip/Reject), batch apply with preview — validated ✓

### Reproduction

- [x] **REPRO-01**: Agent can reproduce issues locally or via SSH using provided configuration
  - Phase 4: ReproductionEngine with exec_shell (local subprocess + asyncssh remote) — validated ✓

### Verification

- [x] **VERIF-01**: Agent can run tests to verify fixes
  - Phase 5: VerificationEngine with pytest --json-report, test file mapping heuristic — validated ✓

- [x] **VERIF-02**: Agent can report verification results
  - Phase 5: verify CLI with Rich display, structured VerificationResult, --output JSON — validated ✓

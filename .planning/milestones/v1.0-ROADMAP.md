# Milestone v1.0: MVP — Ascend Diagnostic Agent

**Status:** ✅ SHIPPED 2026-05-25
**Phases:** 1-5
**Total Plans:** 19

## Overview

First release of the Ascend Diagnostic Agent — an AI agentic system for the Ascend maintenance team to diagnose, fix, reproduce, and verify issues in Python codebases. Covers the full diagnostic pipeline: analyze stack traces → generate hypotheses → suggest fixes → reproduce on test machines → verify with tests.

## Milestone Summary

**Key Decisions:**
- All Pydantic models use ConfigDict(extra="forbid") across all phases
- ModelRouter is concrete class with structured outputs via openai .parse()
- exec_shell reuses asyncio subprocess + asyncssh patterns from Phase 4
- VerificationEngine is deterministic (no LLM) despite accepting ModelRouter for constructor uniformity
- Path traversal protection uses Path.resolve() + startswith() pattern consistently
- All fixes reviewed by human before application (no auto-apply)
- Subprocess approach for test execution (avoids pytest.main() import caching issues)

**Issues Resolved:**
- CLI no-args help via ctx.get_help() (UAT fix)
- MCP server stderr startup banner (stdout is JSON-RPC transport)
- code_search tool name standardization
- exec_shell timeout handling with process kill
- asyncssh Python 3.10+ compatibility constraint

**Known Constraints:**
- Python 3.10+ required (system is 3.9.6 — runtime upgrade needed)
- asyncssh install pending (same Python version constraint)
- VerificationEngine requires pytest-json-report in the target environment
- Full test suite mode deferred (--full flag for future)

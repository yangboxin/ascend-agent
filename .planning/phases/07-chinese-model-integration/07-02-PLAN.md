<objective>
Add structured output fallback to `ModelRouter.completion()` so Chinese providers that lack full `.parse()` support still return valid Pydantic results via manual JSON parsing.

**Purpose:** CHN-04 requires structured output for DeepSeek and Qwen, but both have only partial `.parse()` support. When `.parse()` fails with a 400 error (unsupported `response_format`), fall back to a regular chat completion + `json.loads()` on the content string. The fallback is transparent — all callers continue receiving the same Pydantic model type.

**Output:** Updated `ModelRouter.completion()` with try/except fallback, plus tests mocking both success and failure paths.
</objective>

    Full implementation:
    ```python
    def completion(
        self,
        messages: list[dict],
        response_model: type[BaseModel],
        max_tokens: int = 4096,
        temperature: float = 0.1,
    ) -> BaseModel:
        try:
            completion = self._client.chat.completions.parse(
                model=self._model,
                messages=messages,
                response_format=response_model,
                max_tokens=max_tokens,
                temperature=temperature,
            )
            return completion.choices[0].message.parsed
        except (APIStatusError, BadRequestError) as e:
            if e.status_code != 400:
                raise
            logger.warning(
                "Structured output not supported by provider (model=%s, status=%d). "
                "Falling back to manual JSON parsing.",
                self._model,
                e.status_code,
            )
            completion = self._client.chat.completions.create(
                model=self._model,
                messages=messages,
                max_tokens=max_tokens,
                temperature=temperature,
            )
            content = completion.choices[0].message.content
            if not content:
                raise ValueError(
                    f"Empty response from provider (model={self._model}). "
                    "Cannot parse structured output."
                )
            return response_model.model_validate_json(content)
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Provider response → json.loads | Unvalidated JSON string from provider is parsed with model_validate_json (Pydantic validation applies) |
| 400 error → fallback | Only 400 errors trigger fallback — prevents silent swallowing of auth/rate-limit errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Tampering | fallback JSON parsing | mitigate | model_validate_json provides Pydantic schema validation on the raw content — malformed JSON raises ValidationError |
| T-07-05 | Denial of Service | non-400 passthrough | mitigate | Non-400 errors (auth, rate limit, server errors) propagate immediately — no fallback retry that could mask throttling |
| T-07-06 | Information Disclosure | log message | mitigate | Fallback log includes model name and status code but never message content or API keys |
</threat_model>

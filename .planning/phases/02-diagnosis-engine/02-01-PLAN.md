<action>
Create `src/ascend_agent/diagnosis/router.py` with:

1. A `ModelRouter` class that wraps `openai.OpenAI` client:
   - Constructor takes optional `model: str | None` and `api_key: str | None` params
   - If `api_key` is None, read from `os.environ["OPENAI_API_KEY"]`
   - If key is still None/empty, raise `ValueError("OPENAI_API_KEY is required for diagnosis. Set the OPENAI_API_KEY environment variable.")` — per Pitfall 4 (research.md)
   - If `model` is None, read from `os.environ.get("ASCEND_DIAGNOSIS_MODEL", "gpt-4o")`
   - Store `_client` (OpenAI instance) and `_model` (str) as private attributes
   - NEVER log the api_key value — log `f"ModelRouter initialized (model: {self._model})"` at INFO level

2. A `completion()` method:
   - Signature: `def completion(self, messages: list[dict], response_model: type[BaseModel], max_tokens: int = 4096, temperature: float = 0.1) -> BaseModel`
   - Calls `self._client.chat.completions.parse(model=self._model, messages=messages, response_format=response_model, max_tokens=max_tokens, temperature=temperature)`
   - Returns `completion.choices[0].message.parsed`
   - Per RESEARCH.md: use `.parse()` pattern (not `.create()` + manual parse), Pydantic model as `response_format`

3. A `__repr__` method returning `f"ModelRouter(model={self._model!r})"`

Do NOT create a Protocol/ABC — the ModelRouter is the concrete implementation. A Protocol layer can be added in a later phase if multi-provider routing is needed. Per RESEARCH.md Pattern 2 + discretion area: keep it thin.

Export `ModelRouter` from `src/ascend_agent/diagnosis/__init__.py`.
</action>

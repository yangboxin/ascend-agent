  - Set `language = "python"` (hardcoded per D-12).
  - Return `RepoInfo(path=str(resolved_path), language="python", file_count=file_count, structure=structure)`.

<action>
Create `src/ascend_agent/context/trace.py` with these public functions:

1. `parse_stack_trace(raw_text: str) -> TraceInfo` — Core parser:
  - Use regex `re.compile(r'File "(?P<file>[^"]+)", line (?P<line>\d+)(?:, in (?P<function>\S+))?')` to extract stack frames
  - For each match, create `TraceEntry(file=..., line=int(...), function=..., text=match.group(0))`
  - Extract error type: scan each line for pattern `(?P<type>\w+(?:\.\w+)*)(Error|Exception|Warning|Interrupt|Exit):?\s*(?P<message>.*)`
  - Take the FIRST matching line as the error (typically the last non-empty line in a traceback)
  - If error type is found, set `error_type = match.group("type") + ...` logic per research pattern
  - If no error type found, leave `error_type=None`, `error_message=None`
  - Return `TraceInfo(error_type=..., error_message=..., frames=frames, raw_text=raw_text)`

2. `trace_from_file(path: str | pathlib.Path) -> TraceInfo`:
  - Open file with `encoding="utf-8"`, `errors="replace"`
  - Read content, call `parse_stack_trace()`
  - Return TraceInfo

3. `trace_from_stdin() -> TraceInfo`:
  - Read `sys.stdin` text (blocking read — caller should ensure stdin has content)
  - Call `parse_stack_trace()`
  - Return TraceInfo

4. `trace_from_text(text: str) -> TraceInfo`:
  - Direct call to `parse_stack_trace(text)`
  - Return TraceInfo

Imports: `from ascend_agent.context.models import TraceInfo, TraceEntry`
</action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RepoScanner.scan(path) → filesystem | User-supplied path used for filesystem traversal |
| trace_from_file(path) → filesystem | User-supplied file path read from disk |
| trace_from_stdin() → stdin | Unvalidated text piped into the process |
| trace_from_text(text) → system | Unvalidated inline text processed by regex |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | repo.py (RepoScanner.scan) | mitigate | Resolve path to absolute via `Path.resolve()` — prevents relative path manipulation. D-10 says "trust user's input" (no repo type validation), but path resolution is a basic safety measure, not a project-marker check. |
| T-01-05 | Tampering | trace.py (trace_from_file) | mitigate | Use `Path.resolve()` to get absolute path. Verify file exists and is readable via `Path.is_file()` before opening — catches dangling symlinks and non-file paths early. |
| T-01-06 | Spoofing | trace.py (regex parser) | mitigate | Regex patterns are bounded (no ReDoS in these specific patterns). Pydantic model validation catches malformed TraceEntry/TraceInfo fields. Parsing errors result in `error_type=None` and empty `frames`, not exceptions. |
| T-01-07 | Information Disclosure | repo.py (RepoScanner) | accept | RepoScanner reads file paths and directory structure. No file contents are read. File paths are already visible to the user running the tool locally. Accept — no sensitive data exposure beyond what `ls` shows. |
</threat_model>

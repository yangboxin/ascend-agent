<name>Task 2: Fix MCP Server Startup Message — add stderr output before mcp.run()</name>
<files>src/ascend_agent/tools/server.py</files>
<action>
Add startup feedback to `src/ascend_agent/tools/server.py` by printing the startup status and registered tool names to **stderr** before `mcp.run()` blocks on transport.

Modify the `if __name__ == "__main__":` block:

```python
if __name__ == "__main__":
    import sys

    # Collect registered tool names for startup banner
    tool_names = [t.name for t in mcp._tool_manager.list_tools()]
    print("Starting Ascend Agent MCP server...", file=sys.stderr)
    print(f"Registered tools: {', '.join(sorted(tool_names))}", file=sys.stderr)
    print("Listening on STDIO transport...", file=sys.stderr)
    sys.stderr.flush()
    mcp.run()
```

Key constraints:
- **MUST use `file=sys.stderr`** — stdout is the MCP transport channel. Any print to stdout corrupts the protocol.
- The `import sys` goes at the top of the file OR inside the `if __name__` block. Per Python convention, place `import sys` at file top (line 1) to keep all imports grouped.
- Use `mcp._tool_manager.list_tools()` to get registered tools at runtime (same API the test file uses).
- `sys.stderr.flush()` ensures the startup message appears before `mcp.run()` blocks on I/O.
- Sort tool names alphabetically for consistent output.

The existing tool registrations and their order must remain untouched:
```python
mcp.tool(name="code_search", ...)(search_code)
mcp.tool(name="edit_file", ...)(edit_file)
mcp.tool(name="exec_shell", ...)(exec_shell)
mcp.tool(name="run_test", ...)(run_test)
```
</action>

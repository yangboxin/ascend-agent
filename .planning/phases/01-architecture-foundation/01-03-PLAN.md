Create `run` command:

```python
@diagnose_app.command(name="run")
def diagnose_run(
    repo: str = typer.Argument(..., help="Path to local repository"),
    trace: str | None = typer.Option(None, "--trace", help="Path to trace/log file"),
    trace_text: str | None = typer.Option(None, "--trace-text", help="Inline pasted trace text"),
    output: str | None = typer.Option(None, "--output", help="Path to write context as JSON"),
    interactive: bool = typer.Option(False, "--interactive", "-i", help="Start interactive REPL mode"),
):
    """Analyze a stack trace against a code repository.

    Provide the trace as a file (--trace), inline text (--trace-text), or pipe via stdin.
    The repository path is required and must be a local directory.
    """
```

**One-shot mode** (non-interactive):
  1. Print `[bold]Ascend Diagnostic Agent[/bold]` and `[cyan]Building context...[/cyan]`
  2. Scan repo: `repo_info = RepoScanner().scan(repo)` — wrap in try/except, print `[red]Error:[/red] {e}` if OSError
  3. Parse trace: determine input method priority:
     - `--trace` takes priority: `trace_info = trace_from_file(trace)`
     - else `--trace-text`: `trace_info = trace_from_text(trace_text)`
     - else try stdin: `trace_info = trace_from_stdin()` (only if stdin is not a TTY: `not sys.stdin.isatty()`)
     - else `trace_info = None`
  4. Build `ContextDocument(repo=repo_info, trace=trace_info, config_env=settings)`
  5. Display using `_display_context(document)` function (see below)
  6. If `--output` is specified, write `document.model_dump_json(indent=2)` to the file path

**REPL mode** (`--interactive`):
  1. Print welcome banner: `"[bold]Ascend Diagnostic Agent — REPL mode[/bold]\nType a stack trace or ':help' for commands."`
  2. Scan repo once at startup
  3. Loop: `prompt = console.input("[cyan]>[/cyan] ")` — if input starts with `:`, handle as command:
     - `:help` → list commands
     - `:repo <path>` → rescans with new path
     - `:output` → print JSON of current context
     - `:quit` or `:exit` → break
     - Else → treat as stack trace text, parse, display context

**`_display_context(document: ContextDocument)` helper:**
  - If `document.repo`:
    - Create `Table(title="Repository Info")` with columns "Property" (cyan) and "Value" (green)
    - Add rows: "Path", "Language", "Files", "Structure"
    - For Structure, show first 10 files as comma-separated, with "..." if more
    - `console.print(table)`
  - If `document.trace`:
    - Print `f"\n[bold red]Error:[/bold red] {document.trace.error_type}"`
    - Print `f"[red]{document.trace.error_message}[/red]"`
    - Print `"\n[bold]Stack Trace:[/bold]"`
    - For each frame, display: `f"  [dim]{frame.file}:{frame.line}[/dim] [yellow]{frame.function}[/yellow]"`
    - Max 10 frames shown (with "... N more frames" if truncated)
  - Print `"\n[dim]Environment: Python {version} on {platform}[/dim]"`

  **CRITICAL: Never use `print()` in diagnose.py** — always use `console.print()`. This isolates stdout for potential future MSPC integration (even though the CLI and MCP server are separate processes per the architecture).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI input (file path) → file system | User-supplied diagnosis JSON file path opens a file |
| CLI input (JSON content) → Pydantic model | JSON data crosses into model validation boundary |
| CLI → ReproductionEngine | Trusted model objects flow into engine — no raw input at boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | CLI JSON input | mitigate | JSON is parsed by Pydantic's `model_validate_json()` before any engine call. Pydantic models use `ConfigDict(extra="forbid")` — unknown fields are rejected. Malformed JSON raises JSONDecodeError caught by CLI error handler. |
| T-04-05 | Information Disclosure | CLI --output file | mitigate | ReproductionResult written to disk via `model_dump_json()` — only structured fields, no raw env vars or secrets. The output path is user-controlled; no automatic writing to sensitive locations. |
| T-04-06 | Tampering | CLI file path argument | mitigate | The `diagnosis` argument is a file path — validated by Python's `open()` which rejects directory traversal to a reasonable extent. Pydantic validation further ensures the content matches the DiagnosisOutput schema. |

**T-04-01 disposition: mitigate.** All CLI input flows through Pydantic validation before reaching the engine:
1. File path → `open(diagnosis)` — standard file system access
2. File contents → `DiagnosisOutput.model_validate_json(data)` — validates against schema with extra="forbid"
3. Validated model → `engine.reproduce(diagnosis_output.diagnosis_result)` — only trusted Pydantic objects reach the engine
This defense-in-depth means that even if a malicious JSON file is provided, it must conform to the DiagnosisOutput schema or be rejected before any command execution occurs.

**T-04-05 disposition: mitigate.** The `--output` option writes `result.model_dump_json(indent=2)` — a serialization of the ReproductionResult Pydantic model. This output contains command results (stdout/stderr/exit_code) but no SSH credentials, API keys, or environment variables.
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → env var resolution | User-supplied --provider flag determines which env vars are read |
| env vars → ProviderConfig | Process environment variables flow into Pydantic model then OpenAI client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | create_router / ProviderConfig | mitigate | API keys are read from process env vars, not user input. CLI --provider only selects which env var prefix to read, not the key value itself. |
| T-06-02 | Tampering | env var resolution | mitigate | `os.environ.get()` reads the process environment directly — no file I/O, injection, or parsing risks. |
| T-06-03 | Information Disclosure | ValueError on missing key | accept | Error messages include provider name (e.g., "ASCEND_DEEPSEEK_API_KEY") but never the actual key value. Safe for CLI stderr. |
| T-06-SC | Tampering | no new dependencies | mitigate | Plan 06-01 adds zero new pip/npm packages. All code is stdlib + existing pydantic/openai dependencies. No package gate needed. |
</threat_model>

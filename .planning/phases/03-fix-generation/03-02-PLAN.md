## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → filesystem (read) | fix run reads diagnosis JSON from file or stdin (D-17). JSON is parsed via Pydantic. |
| CLI → filesystem (write) | fix run writes accepted fixes JSON via --output (D-19). |
| CLI → edit_file (call) | Batch apply calls edit_file asyncio.run() — passes through replacement data (D-12). |
| CLI → terminal (interactive) | Rich prompt reads user keyboard input for Accept/Skip/Reject. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | JSON input parsing | mitigate | Pydantic `model_validate_json()` validates structure. Malformed JSON raises JSONDecodeError/ValidationError, caught and printed. |
| T-03-02-02 | Tampering | Stdin input | mitigate | Same as T-03-02-01 — stdin path uses same Pydantic validation. Non-JSON stdin is caught. |
| T-03-02-03 | Spoofing | User input for review | accept | Accept/Skip/Reject prompt uses Rich Prompt.ask with choices=["a","s","r"]. Only valid inputs accepted. Input is not stored or used elsewhere. |
| T-03-02-04 | Tampering | Batch apply via edit_file | mitigate | edit_file has its own path traversal validation (Plan 03-03). CLI passes the repo_path to edit_file. Fix suggestions are validated by FixEngine before reaching CLI. |
| T-03-02-05 | Information Disclosure | --output JSON | accept | Output JSON contains accepted fix suggestions (diff explanations, no secrets expected). User controls output path. |
| T-03-02-SC | Tampering | No new package installs | mitigate | Zero new dependencies — all imports from stdlib (json, sys, os, asyncio), Typer (already installed), Rich (already installed), or this project's own modules. No slopcheck needed. |
</threat_model>

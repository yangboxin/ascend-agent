    Create `src/ascend_agent/diagnosis/fix_engine.py` following the Engine class pattern from `engine.py`.

    The FixEngine class:

    **Constants:**
    - `_FIX_SYSTEM_PROMPT` — module-level string. Instruction: "You are a Python code fixing expert. Given a diagnosis hypothesis with evidence, re-read the relevant code context and generate a precise fix. Output search-and-replace operations (old_text → new_text) where old_text is an exact byte-for-byte match of the original file content. Include enough surrounding context (2-3 lines around the change) in old_text to ensure unique matching. Explain what the fix does and why it addresses the root cause."

    **Constructor `__init__(self, router: ModelRouter, repo_path: str)`:**
    - Store `self._router = router`
    - Store `self._repo_path_resolved = Path(repo_path).resolve()`
    - Identical pattern to Engine.__init__

    **Public method `generate_fixes(self, diagnosis: DiagnosisResult) -> FixGenerationResult`:**
    - Iterate over `diagnosis.hypotheses` (D-05: generate for ALL hypotheses)
    - For each hypothesis, call `self._generate_for_hypothesis(hypothesis, idx)`
    - Collect FixSuggestion objects and PartialFailure errors
    - Return `FixGenerationResult(suggestions=suggestions, errors=errors, total_hypotheses=len(diagnosis.hypotheses))`
    - Wrap each hypothesis iteration in try/except per D-08: "single-shot per hypothesis. LLM reads code, generates one fix. If malformed, note the error and move on."

    **Internal method `_generate_for_hypothesis(self, hypothesis, hypothesis_id: int) -> Optional[FixSuggestion]`:**
    Steps (per RESEARCH.md Pattern 1):
    1. **Read code context** via `self._read_code_context(hypothesis)` — if no code context, log and return None
    2. **Call LLM** via `self._router.completion()`:
       - `messages=[{"role": "system", "content": _FIX_SYSTEM_PROMPT}, {"role": "user", "content": _build_fix_user_prompt(hypothesis, code_context)}]`
       - `response_model=FixResponse`, `max_tokens=4096`, `temperature=0.1`
       - Per D-07: use `ASCEND_FIX_MODEL` env var. Pass `model=os.environ.get("ASCEND_FIX_MODEL", "gpt-4o")` to ModelRouter constructor from the CLI layer (the FixEngine receives the router already constructed; the CLI layer is responsible for constructing ModelRouter with the fix model env var).
    3. **Validate** LLM output:
       - For each Replacement in FixResponse.replacements:
         - Resolve `file_path` relative to `self._repo_path_resolved`
         - Check file exists; if not, log warning and skip
         - Read file and check `old_text in original`; if not found, log warning and skip
         - Check `original.count(old_text) == 1`; if multiple matches, log warning and skip
       - If all replacements are valid, proceed; otherwise return None (D-08: skip on malformed)
    4. **Compute unified diff** for human display:
       - Apply all valid replacements in-memory to compute modified content
       - Use `difflib.unified_diff()`
       - Input: `original.splitlines(keepends=True)` and `modified.splitlines(keepends=True)`
       - `fromfile=file_path, tofile=file_path` (same file path for both since it's the same file)
       - Store the diff string in `diff_patch`
    5. **Return FixSuggestion** with file_path, diff_patch, explanation, hypothesis_id, replacements

    **Internal method `_read_code_context(self, hypothesis) -> str`:**
    - Import `_read_function_body` from `ascend_agent.diagnosis.engine`
    - For each Evidence in hypothesis.evidence:
      - Resolve full path: `self._repo_path_resolved / evidence.file_path`
      - Call `_read_function_body(str(full_path), evidence.line_number, context_lines=5)`
      - Format as `"--- {file_path}:{line_number} ---\n{snippet}"`
    - Join parts with double newlines
    - Return empty string if no context found

    **Helper function `_build_fix_user_prompt(hypothesis, code_context: str) -> str`:**
    - Module-level function (not method on FixEngine)
    - Format: include hypothesis root_cause, evidence details, and code_context
    - Include explicit instruction: "Output search-and-replace operations with old_text being an exact match of the original file content. Do NOT change the old_text content — copy it exactly from the file."

    **Prompt requirements per Pitfall 1 (LLM imprecise search text):**
    - Emphasize "exact byte-for-byte match" for old_text
    - Include explicit example: "old_text must include 2-3 lines of surrounding context for uniqueness"

    **Prompt requirements per Pitfall 2 (duplicate matches):**
    - Instruct LLM to include sufficient surrounding context (at least 2-3 lines around the changed line)

    **Security: Path traversal protection:**
    - In `_generate_for_hypothesis`, before reading any file, verify the resolved path starts with `self._repo_path_resolved` using `Path.resolve()`
    - Skip files that resolve outside the repo with a warning

    **Error handling pattern** (follow engine.py lines 262-280):
    - Catch exceptions in `_generate_for_hypothesis`, log warning, append PartialFailure to FixGenerationResult.errors

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FixEngine → filesystem (read) | FixEngine reads source files from repo_path for code re-reading. Input is self-produced (diagnosis hypotheses reference evidence file paths). |
| diagnose.py → filesystem (write) | `--output` writes DiagnosisOutput JSON to user-specified path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | FixEngine._read_code_context | mitigate | Resolve file paths via `self._repo_path_resolved / evidence.file_path` and validate resolved path starts with repo_path. Skip files outside repo with warning. |
| T-03-01-02 | Information Disclosure | diagnose.py --output | accept | Output JSON contains code context and diagnosis results (non-sensitive). User controls output path. No PII expected. |
| T-03-01-03 | Denial of Service | FixEngine file reads | mitigate | `_read_function_body` in engine.py handles FileNotFoundError/OSError gracefully — FixEngine reuses this. Large files fall back to line-window approach. |
| T-03-01-SC | Tampering | No new package installs | mitigate | Zero new dependencies — all imports are from stdlib (difflib, pathlib) or already-installed packages. No slopcheck needed. |
</threat_model>

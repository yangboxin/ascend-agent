    **Imports:**
    - `import json` (for return dict serialization)

    - `from mcp.server.fastmcp import Context`
    - `from pydantic import BaseModel, ConfigDict, Field` (for EditOperation model)

    **`class EditOperation(BaseModel)` (inline in file_edit.py):**
    - `model_config = ConfigDict(extra="forbid")`
    - `old_text: str` — Field with description: "Exact text to find in the file (byte-for-byte match)"
    - `new_text: str` — Field with description: "Text to replace old_text with"

    **`async def edit_file(file_path: str, operations: list[dict], repo_path: str | None = None, ctx: Context | None = None) -> str`:**
    
    The function signature changes from the current `(path, content)` to `(file_path, operations, repo_path, ctx)` per D-13/D-16.
    
    Parameters:
    - `file_path: str` — Absolute or repo-relative path to the file to edit
    - `operations: list[dict]` — List of {"old_text": str, "new_text": str} dicts
    - `repo_path: str | None` — Repository root path; if provided, validates file_path is within repo (path traversal prevention)
    - `ctx: Context | None` — MCP context for logging

    Implementation steps (per D-13/D-14/D-15/D-16):

    1. **Validate operations input:**
       - Parse each dict into `EditOperation` Pydantic model for validation
       - If parsing fails, return `{"status": "error", "error": "Invalid operation format: ..."}`
       - If operations is empty, return `{"status": "error", "error": "No operations provided"}`

    2. **Path resolution + traversal prevention:**
       - `path = Path(file_path).resolve()`
       - If `repo_path` is provided:
         - `resolved_repo = Path(repo_path).resolve()`
         - Check `str(path).startswith(str(resolved_repo))`
         - If outside repo, return `{"status": "error", "error": f"Path {file_path} resolves outside repository root"}`

    3. **File existence check:**
       - `if not path.exists()`: return `{"status": "error", "error": f"File not found: {file_path}"}`

    4. **Read original file:**
       - `original = path.read_text()` (can raise OSError — catch and return error)

    5. **Validate ALL operations before applying ANY (D-16):**
       - For each operation, check:
         - `if op.old_text not in original`: return error with old_text[:60] truncation
         - `if original.count(op.old_text) > 1`: return error with count and advice to add context
       - If any validation fails, return error immediately — none applied

    6. **Create .bak backup (D-14):**
       - `backup_path = path.with_suffix(path.suffix + ".bak")`
       - If backup_path already exists (safety check), return error mentioning existing .bak
       - `path.rename(backup_path)` — rename original to .bak

    7. **Apply all replacements (D-13, D-15):**
       - `modified = original`
       - For each operation in order: `modified = modified.replace(op.old_text, op.new_text, 1)`
       - Write modified content: `path.write_text(modified)`

    8. **Return success:**
       - `{"status": "ok", "message": f"Applied {len(operations)} replacement(s) to {file_path}", "backup": str(backup_path)}`

    9. **Error handling:**
       - Wrap main logic in try/except
       - For OSError: return `{"status": "error", "error": str(e)}`
       - For any other exception: return `{"status": "error", "error": f"Unexpected error: {e}"}`

    10. **ctx.info logging:**
        - Log at key steps: "Editing {file_path} with {N} replacement(s)", "Backup created at {backup_path}", "Applied {N} replacement(s)"
        - Use `if ctx: await ctx.info(...)` pattern as in code_search.py

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| edit_file (MCP tool) → filesystem | Writes to files in the codebase. Accepts file_path from untrusted callers (LLM-generated fix output). |
| edit_file → .bak files | Creates backup copies during editing. .bak files persist on disk after operation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | edit_file — file_path | mitigate | Validate resolved path starts with repo_path (if repo_path provided). Block paths resolved inside `.git/`. Use `Path.resolve()` check. |
| T-03-03-02 | Tampering | edit_file — old_text injection | mitigate | old_text is checked via plain string `in` operator, not regex. No injection possible. Only verified with `original.count(old_text) == 1` for uniqueness. |
| T-03-03-03 | Information Disclosure | .bak backup files | accept | .bak files contain source code with the same sensitivity as the original. User manages cleanup. Should add `*.bak` to `.gitignore` (if not already present). |
| T-03-03-04 | Denial of Service | edit_file large file | mitigate | `Path.read_text()` reads entire file into memory. Large files (>100MB) will raise OSError, which is caught and returned as error. |
| T-03-03-05 | Tampering | .bak file overwrite | mitigate | Check if backup file already exists (`backup_path.exists()`) before renaming. Return error if `.bak` already exists. |
| T-03-03-SC | Tampering | No new package installs | mitigate | Zero new dependencies. All imports from `pathlib` (stdlib), `json` (stdlib), `mcp` (already installed), `pydantic` (already installed). No slopcheck needed. |
</threat_model>

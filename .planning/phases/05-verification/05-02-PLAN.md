**Imports (top-level):**

**Class: VerificationEngine:**

Constructor signature (exact match to ReproductionEngine):
```python
def __init__(self, router: ModelRouter, repo_path: str, settings: Settings | None = None):
    self._router = router
    self._repo_path = Path(repo_path).resolve()
    self._settings = settings or Settings()
```

**Public async method: verify(reproduction: ReproductionResult) → VerificationResult:**

Workflow: detect framework → map test files → build command → execute via exec_shell → parse JSON → return VerificationResult.

1. **Empty files_changed early return (D-02 edge case):** if not reproduction.files_changed, return VerificationResult(status="no_tests", hypothesis_id_verified=reproduction.hypothesis_id_tested, framework=None, command="", summary="No files were changed during reproduction — nothing to verify.", tests_found=0, tests_run=0, passed=0, failed=0, errors=0, skipped=0, xfailed=0, xpassed=0, tests=[], exit_code=-1, duration_seconds=0.0, files_tested=[])

2. **Detect framework (D-01):** call self._detect_framework(). If returns None, return VerificationResult(status="error", framework=None, command="", summary="No supported test framework detected in the repository. Supported: pytest (looks for pytest.ini, pyproject.toml [tool.pytest.ini_options], setup.cfg [tool:pytest])", ...)

3. **Map test files (D-02):** call self._map_test_files(reproduction.files_changed). If returns empty list, return VerificationResult(status="no_tests", summary=f"No relevant test files found for changed files: {', '.join(reproduction.files_changed)}", tests_found=0, tests_run=0, ...)

4. **Build command (D-03):** construct pytest command — `cd {self._repo_path} && python -m pytest {test_files joined by space} --json-report --json-report-file=none --json-report-summary`

5. **Execute via exec_shell (D-04):** lazy import inside method body like ReproductionEngine: `from ascend_agent.tools.shell_exec import exec_shell`. Call `exec_shell(command, timeout=self._settings.test_timeout)`. Measure duration with time.monotonic().

6. **Parse JSON:** json.loads(result_json). Extract from report dict:
   - exit_code from exec_shell result
   - summary counts: report["summary"]["passed"], report["summary"]["failed"], report["summary"]["error"], report["summary"]["skipped"], report["summary"]["xfailed"], report["summary"]["xpassed"]
   - tests list: report.get("tests", []) — map each to TestDetail(nodeid=t["nodeid"], outcome=t["outcome"], duration=t.get("duration"), message=t.get("message"))

7. **Build VerificationResult:** determine status: "pass" if exit_code==0 and failed==0 and errors==0, "timeout" if stderr contains "timed out after", "fail" if exit_code!=0, "error" if exception caught

8. **Log:** logger.info("Verification complete: %d/%d passed (%.2fs)", passed, passed+failed+errors, duration)

9. **Exception handling (wrap entire try/except):** on any Exception, logger.error("Verification failed: %s", exc), return VerificationResult(status="error", command=command, summary=f"Verification failed: {exc}", stderr=str(exc), exit_code=-1, duration_seconds=0.0, hypothesis_id_verified=reproduction.hypothesis_id_tested, framework=framework, ...)

**Private method: _detect_framework() → str | None (D-01):**
Check target repo for pytest config files in order:
1. pytest.ini: (self._repo_path / "pytest.ini").exists()
2. pyproject.toml with [tool.pytest.ini_options]: read file, check for "[tool.pytest.ini_options]" or "tool.pytest" section (use simple string search, not TOML parser)
3. setup.cfg with [tool:pytest]: read file, check for "[tool:pytest]"
Return "pytest" on first match. Return None if no config found.

**Private method: _map_test_files(files_changed: list[str]) → list[str] (D-02):**
Three-tier heuristic:
1. Exact convention match: for each file in files_changed, strip src/ prefix, prepend tests/, change filename to test_{original}:
   - src/ascend_agent/tools/shell_exec.py → tests/test_tools/test_shell_exec.py (but simpler: strip src/, replace / with /test_ in the last component)
   - Implementation: Path(f).stem gives "shell_exec", then tests/{parent}/{test_stem}.py
   - Check if resolved path exists inside repo using _validate_path and Path.exists()
2. Glob search: if exact match doesn't exist, look for *_test.py and test_*.py variants in corresponding test dir
3. Return deduplicated list of existing test file paths (repo-relative)

**Private method: _validate_path(path_str: str) → bool:**
Copy verbatim from ReproductionEngine (lines 166–175): Path.resolve() + startswith(self._repo_path). Also used for validating test file paths mapped by the heuristic.

**Private method: _detect_pytest_json_report() → bool:**
Check if pytest-json-report is available by attempting: python -c "import pytest_jsonreport" via asyncio subprocess. Returns True/False. If False, include warning in VerificationResult summary.
